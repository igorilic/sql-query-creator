# SQL Query Creator — Technical Specification

**ID:** feature-20260320-120720
**Date:** 2026-03-20
**Requirements:** `.context/specs/requirements.md`

---

## 1. Problem Statement

Developers need a local-first tool that converts plain English descriptions into schema-aware SQL queries using a local LLM (LM Studio). The tool connects to PostgreSQL or SQLite databases, introspects their schema, and uses that context to produce accurate, dialect-appropriate SQL.

---

## 2. Context

- **Existing assets:** Catalyst UI Kit (TypeScript components) already vendored at `catalyst-ui-kit/typescript/` — includes SidebarLayout, Input, Select, Button, Textarea, Badge, and 20+ components built on HeadlessUI + Tailwind CSS v4.
- **Stack:** Next.js 16 + TypeScript + Tailwind CSS v4 + React 19.
- **Monorepo:** pnpm workspaces — `apps/web/` (Next.js), `packages/ui/` (Catalyst components).
- **LLM:** LM Studio local API (OpenAI-compatible at `http://localhost:1234/v1`).
- **No existing app code** — greenfield project.

### 3.2 Component Design

**Layout:** `SidebarLayout` from Catalyst — sidebar holds schema browser, main content area holds chat + editor in a vertical split.

**Pages (App Router):**
- `/` — Main page (single-page app feel). Connection dialog opens as a modal.

**API Routes:**
| Route | Method | Purpose |
|-------|--------|---------|
| `/api/connect` | POST | Establish DB connection, return status |
| `/api/schema` | GET | Return introspected schema (tables, columns, types, FKs) |
| `/api/chat` | POST | Stream LLM response via SSE (sends schema + history + user message) |

**State Management:** React Context + `useReducer` for:
- `ConnectionContext` — connection status, db type, schema data
- `ChatContext` — messages array, loading state, current SQL

**Key Libraries:**
| Library | Purpose |
|---------|---------|
| `pg` | PostgreSQL client |
| `better-sqlite3` | SQLite client (synchronous, fast) |
| `@codemirror/lang-sql` | SQL syntax highlighting in editor |
| `codemirror` | Code editor component |

### 3.3 Data Models

```typescript
// Schema types
interface DatabaseSchema {
  tables: TableInfo[];
  dialect: 'postgresql' | 'sqlite';
}

interface TableInfo {
  name: string;
  columns: ColumnInfo[];
}

interface ColumnInfo {
  name: string;
  dataType: string;
  nullable: boolean;
  isPrimaryKey: boolean;
  foreignKey?: { table: string; column: string };
}

// Chat types
interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  sql?: string; // Extracted SQL block, if any
  timestamp: number;
}

// Connection types
interface ConnectionConfig {
  type: 'postgresql' | 'sqlite';
  // PostgreSQL
  host?: string;
  port?: number;
  database?: string;
  username?: string;
  password?: string;
  // SQLite
  filePath?: string;
}

interface ConnectionStatus {
  connected: boolean;
  type?: 'postgresql' | 'sqlite';
  error?: string;
}
```

### 3.4 LLM Prompt Strategy

System prompt template:
```
You are a SQL expert. Generate syntactically correct {dialect} SQL based on the user's request.
{schema_context}  // Only included when DB is connected
Return SQL in a ```sql code block. Explain your reasoning briefly.
```

Schema context format:
```
Database schema ({dialect}):
Table: users
  - id (integer, PK)
  - email (varchar, NOT NULL)
  - created_at (timestamp)
Table: orders
  - id (integer, PK)
  - user_id (integer, FK → users.id)
  ...
```

### 3.5 SSE Streaming

The `/api/chat` route streams tokens from LM Studio using Server-Sent Events. The client accumulates tokens, parses SQL blocks in real-time, and updates both the chat panel and query editor as the response arrives.

## 5. Risks

| Risk | Mitigation |
|------|------------|
| LM Studio API may be slow or unavailable | Graceful error handling + timeout (30s). App fully usable without LLM. |
| `better-sqlite3` requires native compilation | Use prebuild binaries; document Node.js version requirement. |
| Large schemas may exceed LLM context window | Truncate schema to most relevant tables (future: let user select tables). For now, send full schema with a 200-table cap. |
| Catalyst UI Kit may need modifications | Copy into `packages/ui/`, treat as internal code. |

## 6. Open Questions — Resolved Assumptions

| OQ | Decision | Rationale |
|----|----------|-----------|
| OQ-1 | LM Studio URL configurable via `LMSTUDIO_BASE_URL` env var only. | Keeps MVP simple; settings UI is out of scope. |
| OQ-2 | Streaming (SSE) is implemented. | Critical for UX — A-5 in requirements. |
| OQ-3 | Max 20 messages in conversation history. Oldest trimmed. | Prevents context overflow while keeping useful history. |
| OQ-4 | FK relationships shown as text badges on columns, not visual arrows. | Simpler implementation; arrows can be added later. |
| OQ-5 | System prompt is hardcoded. | Out of scope for MVP. |
| OQ-6 | Catalyst UI Kit copied into `packages/ui/` as source files. | Matches existing repo structure (A-3). |

## 7. Out of Scope

- Query execution and result display
- Query history / saved queries
- Multiple simultaneous database connections
- Model selection within the app
- Databases other than PostgreSQL and SQLite
- User authentication
- Schema diff / migrations
- Cloud deployment
- Settings UI for LM Studio URL
- Customizable system prompts
